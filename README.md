# sparkle-addons-extra
`sparkle-addons-extra` is a collection of [Sparkle](https://github.com/Mojavesoft-Group/sparkle) addons that are held to lower standards than those in the [SparkleAddons](https://github.com/Mojavesoft-Group/SparkleAddons) repository.

To download from it in Sparkle v0.8 and later, install the _Custom Repositories_ addon (available in SparkleAddons) and set your repository URL to `https://raw.githubusercontent.com/Mojavesoft-Group/sparkle-addons-extra/refs/heads/master/`.

## Advice for end-users
Unless you know what you're doing, it's unwise to obtain addons from `sparkle-addons-extra`. Addons found in this repository may contain serious flaws or contain code of an especially-poor quality, and the Sparkle developers are not responsible for any damage caused by the usage of these addons.

Most end-users should use the [SparkleAddons](https://github.com/Mojavesoft-Group/SparkleAddons) collection instead, because it has higher-quality software that is held to much stricter standards.

## Rules for submission
All addons in this repository must meet at least one of the following criteria:
- the addon is written partially or entirely with an LLM;
- the addon is in early development;
- the addon contains non-free code or media, under license;
- the addon is of suspect quality; or
- the addon is no longer actively-maintained.

 We also won't accept any of the following:
- addons of an inherently-malicious nature (for example, an addon that seeks to lock the user out of their own account)
- addons which have little or no apparent practical purpose (for example, a "Hello World" addon)
- addons whose publication in this repository amounts to intellectual property infringement (for example, an addon that contains pirated code from another project)
- addons that only serve to duplicate another addon's functionality (for example, a clone of the "Jameson Logo" addon that uses a different license)
- addons that don't have a clearly-specified software license somewhere in their code

Addons that fail to meet the current criteria may be removed without notice. Reviewers have full discretion in accepting or rejecting pull requests.
